## 🤝 협업 방식 및 병합 방법
### 브랜치 전략
- `main` : 안정적인 배포 버전
- `develop` : 개발 브랜치
- `feature/{기능명}` : 새로운 기능 개발
- `hotfix/{수정명}` : 긴급 수정 사항

### 병합 방법 
1. 이슈를 확인하고 원하는 기능을 선택합니다.
2. 새로운 브랜치를 생성합니다.
```sh
git checkout -b feature/new-feature
```
3. 코드를 작성하고 커밋합니다.
```sh
git commit -m "✨ 새로운 기능 추가"
```
4. 원격 저장소에 푸시하고 PR을 생성합니다.
```sh
git push origin feature/new-feature
```
5. 코드 리뷰 후 `develop` 브랜치에 병합합니다.
